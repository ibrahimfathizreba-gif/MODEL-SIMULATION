## 1. الملخص التنفيذي

هذا المشروع عبارة عن **منصة محاكاة اقتصادية متكاملة** صُممت للإجابة عن سؤال محوري واحد:

> **"كيف سيغيّر الذكاء الاصطناعي شكل سوق العمل العالمي خلال العشرين سنة القادمة (2026 – 2046)؟"**

المشروع لا يقدّم إجابة واحدة جامدة، بل يبني **نموذجاً ديناميكياً متعدد الطبقات** يحاكي:

- **سوق العمل** (الوظائف، البطالة، القوى العاملة)
- **القطاعات الاقتصادية الأربعة** (التقنية، التصنيع، الصحة، الخدمات)
- **خمسة مستويات من المهارات** (من العامل غير الماهر إلى الخبير المتقدم)
- **منحنى تبنّي الذكاء الاصطناعي** (S-Curve لوجستي)
- **التأثيرات الاقتصادية الكلية** (الناتج المحلي، الأجور، عدم المساواة، الإنفاق الاستهلاكي)

النظام يجمع بين:
- **35 دالة رياضية** للمحاكاة العددية
- **نموذج تعلم آلي (Random Forest)** للتنبؤ بمعدلات البطالة
- **محاكاة مونت كارلو** (1000+ تكرار) لقياس عدم اليقين
- **تحليل الحساسية** لقياس تأثير تغيير المتغيرات
- **تكامل مع Claude AI** لتوليد تقرير استشاري اقتصادي احترافي باللغة العربية

النتيجة: **لوحة تحكم تفاعلية احترافية** تعرض البيانات في 6 صفحات بصرية متخصصة، مدعومة بـ API احترافي مبني بإطار FastAPI.

## 2. الفكرة الجوهرية والرؤية

### 2.1 المشكلة التي يحلّها المشروع

العالم يقف اليوم على أعتاب تحوّل اقتصادي ضخم بفعل الذكاء الاصطناعي. الأسئلة التي تطرحها الحكومات والمنظمات الدولية وشركات التوظيف:

- كم وظيفة ستفقد؟ وكم وظيفة جديدة ستُخلَق؟
- أي القطاعات الأكثر عرضة للأتمتة؟
- كيف ستتأثر الأجور حسب مستوى المهارة؟
- هل ستزيد الفجوة بين الأغنياء والفقراء؟
- ما السياسات المطلوبة لإعادة تدريب العمال المتضررين؟

التقارير التقليدية تقدّم **توقعات ثابتة**، لكن الواقع يتطلّب **أداة استكشافية** يستطيع متّخذ القرار من خلالها تجريب سيناريوهات مختلفة ورؤية النتائج فوراً.

### 2.2 الحل المُقدَّم

المشروع يقدّم **مختبراً افتراضياً للسياسات الاقتصادية** يسمح للمستخدم بـ:

1. **اختيار سيناريو** (تبنّي بطيء / متوسط / سريع للذكاء الاصطناعي)
2. **تعديل المتغيرات** عبر منزلقات تفاعلية (سرعة التبنّي، معدّل الأتمتة، الأفق الزمني)
3. **مشاهدة النتائج** على شكل رسوم بيانية تفاعلية فورية
4. **الحصول على تقرير استشاري** مكتوب بالعربية من Claude AI

### 2.3 الفلسفة وراء التصميم

- **شفافية كاملة**: كل دالة رياضية مُوثّقة ومفهومة
- **بيانات حقيقية**: استخدام بيانات البنك الدولي ومنظمة العمل الدولية ودراسات أكسفورد
- **تعدّد الطبقات**: لا يكفي حساب البطالة فقط — يجب ربطها بالقطاعات والمهارات والأجور
- **قابلية التحقق**: النموذج يُختبر ضد بيانات تاريخية (2000-2020) ويُعطي درجة دقة قابلة للقياس

### 2.4 الجمهور المستهدف

- **منظمات دولية** (منظمة العمل الدولية، البنك الدولي، صندوق النقد)
- **الحكومات ووزارات العمل والاقتصاد**
- **الشركات الكبرى** للتخطيط طويل الأمد للقوى العاملة
- **مراكز الأبحاث والجامعات**
- **المستثمرون** الذين يتابعون تأثير الذكاء الاصطناعي على الاقتصاد

### 3.3 تدفّق البيانات النموذجي

عند ضغط المستخدم على زر "Run Simulation":

1. **المتصفح** يرسل طلب `GET /api/simulate/moderate?horizon=20&adoption_speed=1.0`
2. **FastAPI** يستقبل الطلب ويتحقق من صحته
3. **simulation.py** يبني الحالة الابتدائية للسوق ويبدأ حلقة 20 سنة
4. كل سنة: تُحدّث الوظائف، القوى العاملة، البطالة، الناتج المحلي، توزيع المهارات، توزيع القطاعات، الأجور، مؤشّر جيني
5. **ml_forecaster.py** يدرّب نموذج Random Forest على بيانات تاريخية ويتنبّأ بمعدل البطالة
6. النتائج تُجمَع في كائن JSON واحد وتُرسَل للمتصفح
7. **ApexCharts** يرسم الرسوم البيانية ويُحدّث بطاقات KPI

كل هذا يحدث في **أقل من ثانيتين**.

## 6. الواجهة الخلفية — Backend

### 6.1 ملف `backend/app.py` — قلب الـ API

هذا الملف هو **نقطة دخول النظام**. يُعرّف 6 نقاط نهاية (endpoints) ويربطها بالدوال الموجودة في الملفات الأخرى.

#### بنية الملف:

```python
from fastapi import FastAPI, HTTPException, Body
from fastapi.middleware.cors import CORSMiddleware
from pydantic import BaseModel

app = FastAPI(
    title="AI Labor Market Impact Simulation API",
    version="1.0.0",
)

# تفعيل CORS للسماح للواجهة الأمامية بالاتصال
app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_methods=["*"],
    allow_headers=["*"],
)
```

#### نماذج Pydantic (للتحقق من المدخلات):

```python
class SimulateParams(BaseModel):
    horizon:        Optional[int]   = 20
    adoption_speed: Optional[float] = 1.0
    export_csv:     Optional[bool]  = False

class AnalyzeRequest(BaseModel):
    scenario:       Optional[str]   = "moderate"
    horizon:        Optional[int]   = 20
    adoption_speed: Optional[float] = 1.0
```

#### النقاط الست (Endpoints):

| الطريقة | المسار | الوظيفة |
|---------|--------|---------|
| `GET` | `/` | فحص جاهزية الـ API |
| `GET` | `/api/simulate/{scenario}` | تشغيل محاكاة كاملة لـ 20 سنة |
| `GET` | `/api/validate` | اختبار النموذج على بيانات 2000-2020 |
| `GET` | `/api/monte-carlo` | محاكاة مونت كارلو (1000+ تكرار) |
| `GET` | `/api/compare` | مقارنة جميع السيناريوهات الثلاثة |
| `GET` | `/api/sensitivity` | تحليل الحساسية للمتغيرات |
| `POST` | `/api/analyze` | تشغيل محاكاة + توليد تقرير Claude AI |

#### مثال على نقطة نهاية تفصيلية:

```python
@app.get("/api/simulate/{scenario}")
def simulate(
    scenario:       str,
    horizon:        int   = 20,
    adoption_speed: float = 1.0,
    export_csv:     bool  = False,
):
    """تشغيل محاكاة كاملة لمدة 20 سنة"""
    valid_scenarios = {"slow", "moderate", "rapid"}
    if scenario not in valid_scenarios:
        raise HTTPException(status_code=400, ...)
    try:
        results = run_scenario(scenario, horizon=horizon,
                              adoption_speed=adoption_speed)
        results["report"] = generate_report(results)

        # إضافة تنبؤ التعلم الآلي
        ml_data = forecast_unemployment(2025, 2025 + horizon - 1)
        results["ml_forecast"] = ml_data

        return results
    except Exception as e:
        raise HTTPException(status_code=500, detail=...)
```

**الملاحظات على هذا الملف:**
- معالجة الأخطاء شاملة (try/except لكل endpoint)
- التحقق من السيناريوهات الصالحة قبل التنفيذ
- حماية من إفراط مونت كارلو (clamp بين 100 و 5000 تكرار)
- إرجاع رسائل خطأ واضحة بصيغة HTTP standard

## 7. النموذج الرياضي للمحاكاة

ملف `simulation.py` يحتوي على **9 مجموعات منطقية** من الدوال. كل مجموعة تمثّل طبقة من طبقات الاقتصاد.

### 7.1 المجموعة 1 — نموذج السوق الأساسي (Basic Market Model)

**الهدف:** تتبّع أعداد الوظائف والقوى العاملة وحساب البطالة.

#### الدوال:

##### `initialize_market()` — تهيئة الحالة الأولية لعام 2026

```python
{
    "base_year":       2026,
    "total_jobs":      150_000_000,    # 150 مليون وظيفة
    "workforce":       160_000_000,    # 160 مليون قوة عاملة
    "gdp_base":        25.0,           # 25 تريليون دولار
    "ai_adoption":     0.05,           # 5% تبنّي مبدئي
    "automation_rate": 0.03,           # 3% معدّل الأتمتة
}
```

##### `update_jobs(state, year, automation_rate)` — تحديث الوظائف سنوياً

```python
jobs_lost     = total_jobs × automation_rate × ai_adoption
jobs_created  = new_ai_jobs_created(state, automation_rate)
total_jobs    = max(0, total_jobs − jobs_lost + jobs_created)
```

**الفكرة:** كل سنة، نسبة من الوظائف تُؤتمت (تختفي)، لكن في نفس الوقت تُخلَق وظائف جديدة بفضل الذكاء الاصطناعي.

##### `update_workforce(state, year)` — نمو السكان

```python
workforce *= (1 + 0.008)   # نمو سنوي 0.8%
```

##### `calculate_unemployment(state)` — معدل البطالة

```python
unemployment = max(0, (workforce − jobs) / workforce)
```

**الإخراج:** عدد الوظائف المفقودة، الوظائف المخلوقة، الحالة المُحدَّثة، معدل البطالة.

### 7.2 المجموعة 2 — مستويات المهارات (Skill Tiers)

**الهدف:** تمثيل الواقع — العمال ليسوا متجانسين. هناك 5 مستويات من المهارة، وكل مستوى يواجه مخاطر مختلفة من الأتمتة.

#### التوزيع الافتراضي:

| المستوى | الوصف | النسبة |
|---------|-------|--------|
| **L1_basic** | مهارات أساسية (وظائف يدوية متكرّرة) | 20% |
| **L2_semi** | نصف ماهر | 30% |
| **L3_intermediate** | متوسط (موظف عادي) | 25% |
| **L4_advanced** | متقدم (مدير / مهندس) | 15% |
| **L5_expert** | خبير (بحث وتطوير / تخصص نادر) | 10% |

#### مخاطر الأتمتة لكل مستوى:

```python
risk = {
    "L1_basic":         0.75,   # 75% من وظائفهم معرّضة للأتمتة
```

### 7.5 المجموعة 5 — ديناميكيات الذكاء الاصطناعي (AI Dynamics)

**الهدف:** نمذجة كيف ينتشر الذكاء الاصطناعي في الاقتصاد.

#### `ai_adoption_curve(year_index, speed=1.0)` — منحنى S اللوجستي

```python
k  = 0.35 × speed   # حدة المنحنى
x0 = 10.0           # نقطة الانعطاف (السنة 10)
return 1 / (1 + exp(−k × (year_index − x0)))
```

**هذا هو منحنى التبنّي الكلاسيكي:**
- بداية بطيئة (السنوات 1-5)
- تسارع كبير (السنوات 6-15)
- تشبّع (السنوات 16-20)

## 8. نموذج التعلم الآلي — ML Forecaster

ملف `backend/ml_forecaster.py` يستخدم **Random Forest Regressor** للتنبؤ بمعدل البطالة.

### 8.1 لماذا Random Forest؟

- **يتعامل مع البيانات غير الخطية** (وسوق العمل غير خطي)
- **مقاوم للضوضاء** (overfitting أقل من Decision Tree واحدة)
- **يعطي feature importances** (نعرف أي المتغيرات أهم)
- **سريع التدريب** (مناسب للتشغيل الفوري في API)

### 8.2 هندسة الميزات (Feature Engineering)

من البيانات الخام يُنشئ النموذج الميزات التالية:

```python
df["lag1"]     = previous year's unemployment
df["lag2"]     = 2 years ago
df["lag3"]     = 3 years ago
df["trend"]    = year - 2000   (متغير الزمن)
df["rolling3"] = average of last 3 years
df["ai_proxy"] = max(0, (year − 2015)² / 100)   # نمو أُسي تقريبي للذكاء الاصطناعي
```

**فكرة `ai_proxy`:** قبل 2015 = 0، ثم ينمو تربيعياً ليُحاكي تسارع تبنّي الذكاء الاصطناعي.

### 8.3 إعدادات النموذج

## 13. طرق التحقق من صحة النموذج

النظام يوظّف **ثلاث طبقات من التحقق** — وهذا ما يجعله ذا مصداقية علمية.

### 13.1 الاختبار التاريخي (Backtesting)

**الفكرة:** نُشغّل النموذج على الفترة 2000-2020 ونقارن تنبؤاته بالبيانات الفعلية من البنك الدولي.

**المقاييس:**
- **MAE (Mean Absolute Error):** متوسّط الخطأ المطلق
- **Accuracy:** `max(0, 100 − MAE × 10)`

**النتيجة النموذجية:** دقة ~85-95% (تتفاوت حسب توفّر بيانات الـ API).

### 13.2 محاكاة مونت كارلو

**الفكرة:** بدلاً من رقم واحد، نُولّد 1000 سيناريو عشوائي ونحسب التوزيع الإحصائي.

**القيم العشوائية:**
- `automation_rate`: عشوائي بين 2% و 10%
- `adoption_speed`: عشوائي بين 0.6 و 1.5
- `initial jobs/workforce`: تذبذب ±5%

**المخرجات:**
- المتوسط (mean)
- الحد الأدنى لفترة الثقة 95% (2.5th percentile)
- الحد الأعلى لفترة الثقة 95% (97.5th percentile)

**هذا يُعطي للمستخدم فكرة عن "عدم اليقين" في التنبؤات — وهي ممارسة قياسية في الاقتصاد القياسي.**

### 13.3 تحليل الحساسية (Sensitivity Analysis)

**الفكرة:** نُغيّر كل متغير بنسبة ±20% ونقيس مدى تغيّر النتيجة النهائية.

**المخرجات:**
- إذا تغيّر `automation_rate` بـ +20% → كم تتغيّر البطالة النهائية؟
- إذا تغيّر `adoption_speed` بـ −20% → كم يتغيّر الناتج المحلي؟

**الفائدة:** نعرف **أي المتغيرات الأكثر تأثيراً** على النتائج — وهذا حاسم لصنّاع السياسات.
