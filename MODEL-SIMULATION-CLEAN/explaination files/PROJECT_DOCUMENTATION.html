§1 Executive Summary

This project simulates the impact of AI adoption on labor markets over a 20-year horizon (2026–2046). Users select a country, an AI-rollout scenario (slow/moderate/rapid), and an adoption-speed multiplier; the system returns year-by-year projections of unemployment, GDP, productivity, sector composition, skill-tier wages, and income inequality.

20 Functions in simulation.py
11 REST API endpoints
6 Interactive dashboard pages
30 Countries supported
7 Live World Bank indicators
1,000 Monte Carlo runs per query

Key features

  • Country-level simulation — 30 World Bank country codes, with live baseline data fetched at runtime.
  • Three scenarios — slow (3%), moderate (5%), and rapid (8%) base annual automation rates.
  • Five skill tiers — calibrated to BLS Occupational Employment Wage Statistics (May 2024).
  • Four sectors — Tech, Manufacturing, Healthcare, Services, with cross-sector spillover.
  • Monte Carlo uncertainty — 1,000 simulations with parameter jitter, returns 95% confidence intervals.
  • Sensitivity analysis — ±20% perturbation on each parameter to identify high-leverage inputs.
  • Historical backtest — model is validated against World Bank unemployment data 2000–2020.
  • Random Forest forecaster — independent ML projection trained on 2000–2024 unemployment.
  • AI-generated advisory report — Arabic economic commentary via Ollama (local LLM).
  • Frey & Osborne 2013 integration — 50+ occupations' automation probabilities, sourced from Oxford Martin School.

ai_adoption(year_index, speed) lines 158–161 · the S-curve driver

Logistic S-curve. Returns AI adoption as a fraction from 0 to 1. Midpoint at year 10, controlled by speed.

Code
def ai_adoption(year_index: int, speed: float = 1.0) -> float:
    """Logistic S-curve, midpoint at year 10, returns 0..1."""
    k = 0.35 * speed
    return 1.0 / (1.0 + np.exp(-k * (year_index - 10.0)))
Logic flow
flowchart TD A[year_index, speed] --> B["k = 0.35 × speed"] B --> C["Compute logistic:
1/(1 + e^(-k(year-10)))"] C --> D[Return 0..1] style D fill:#238636,color:#fff

Output values (speed=1.0): year 0 → 3%, year 5 → 15%, year 10 → 50%, year 15 → 85%, year 20 → 97%.

step_year(state, year_index, auto_rate, speed) lines 175–192 · the yearly engine

Advances the macro state by one year. Updates adoption, jobs, workforce, and computes unemployment, productivity, GDP, and consumer spending.

Code
def step_year(state, year_index, auto_rate, speed):
    state["ai_adoption"] = ai_adoption(year_index, speed)
    adoption = state["ai_adoption"]

    jobs_lost    = state["total_jobs"] * auto_rate * adoption
    jobs_created = state["total_jobs"] * auto_rate * adoption * JOB_CREATION_RATIO
    state["total_jobs"] = max(0.0, state["total_jobs"] - jobs_lost + jobs_created)
    state["workforce"] *= (1.0 + WORKFORCE_GROWTH)

    gdp_base    = state.get("_gdp_base", INITIAL_GDP)
    unem        = max(0.0, (state["workforce"] - state["total_jobs"]) / state["workforce"])
    productivity= 1.0 + (adoption ** 0.7) * 0.85
    gdp         = gdp_base * (1.0 + (productivity - 1.0) * 0.6 - unem * 0.4)
    spending    = gdp * max(0.30, 0.68 - unem * 1.2)

    return {"unem": unem, "productivity": productivity,
            "gdp": gdp, "spending": spending}

§5 Backend: forecast.py — Random Forest Forecaster

Independent machine-learning baseline. Trains a Random Forest Regressor on World Bank global unemployment (2000–2024) and projects forward year by year, feeding each prediction back as a lagged feature. Serves as a sanity-check counterpart to the macroeconomic model.

5.1 Feature engineering

Six features per year:

FeatureFormulaWhat it captures
trendyear − min_yearLong-run linear trend (years since 2000)
lag1unem(t−1)Last year's unemployment (autoregressive)
lag2unem(t−2)Two years ago
lag3unem(t−3)Three years ago
rolling3mean of last 3 yearsSmoothed local level

5.2 Functions

_build_features(years, values) lines 23–31

Constructs the feature DataFrame from raw time series. Drops the first 3 rows that have NaN lags.

Code
def _build_features(years, values) -> pd.DataFrame:
    df = pd.DataFrame({"year": years, "unemployment_rate": values}) \
           .sort_values("year").reset_index(drop=True)
    df["lag1"]     = df["unemployment_rate"].shift(1)
    df["lag2"]     = df["unemployment_rate"].shift(2)
    df["lag3"]     = df["unemployment_rate"].shift(3)
    df["trend"]    = df["year"] - df["year"].min()
    df["rolling3"] = df["unemployment_rate"].rolling(3).mean()
    df["ai_proxy"] = df["year"].apply(lambda y: max(0.0, (y - 2015) ** 2 / 100.0))
    return df.dropna().reset_index(drop=True)

train() lines 34–60

Pulls 25 years of WB unemployment data, builds features, trains a Random Forest, and returns the fitted model along with its mean absolute error.

Code
def train() -> Dict[str, Any]:
    raw = fetch_world_bank_unemployment(2000, 2024)
    df  = _build_features(raw["years"], raw["values"])

    X = df[FEATURE_COLS].values
    y = df["unemployment_rate"].values

    scaler = StandardScaler()
    X_scaled = scaler.fit_transform(X)

    model = RandomForestRegressor(
        n_estimators=200, max_depth=8,
        min_samples_leaf=2, random_state=42, n_jobs=-1,
    )
    model.fit(X_scaled, y)
    mae = mean_absolute_error(y, model.predict(X_scaled))

    return {"model": model, "scaler": scaler,
            "mae": mae, "accuracy": 100 - mae * 10,
            "base_year": int(df["year"].min())}

forecast_unemployment(start, end) lines 63–94

Generates an autoregressive forecast: each year's prediction becomes a lag feature for the next year. This is the function called by the /api/simulate endpoint to attach an ML-based comparison line to every simulation.

Code (abridged)
def forecast_unemployment(start=2025, end=2044):
    trained = train()
    model, scaler = trained["model"], trained["scaler"]
    raw = fetch_world_bank_unemployment(2000, 2024)
    history = list(raw["values"][-5:])
    base_year = trained["base_year"]

    years, predicted = list(range(start, end + 1)), []
    for year in years:
        lag1 = history[-1]; lag2 = history[-2]; lag3 = history[-3]
        rolling3 = np.mean(history[-3:])
        feats = np.array([[year - base_year, lag1, lag2, lag3, rolling3,
                            max(0, (year-2015)**2 / 100)]])
        pred = float(model.predict(scaler.transform(feats))[0])
        pred = round(max(0.0, pred), 3)
        predicted.append(pred)
        history.append(pred)  # autoregressive feedback

    importances = {col: float(v) for col, v in zip(FEATURE_COLS, model.feature_importances_)}
    return {"years": years, "predicted_unemployment": predicted,
            "model_accuracy": trained["accuracy"],
            "model_mae": trained["mae"],
            "feature_importances": importances}

§6 Backend: llm.py — AI Advisory Report (Arabic)

Generates a 6-section economic advisory report in formal Modern Standard Arabic, using a local Ollama LLM. Designed for the project's Arabic-speaking audience. Default model is kimi-k2.6:cloud (1T params, cloud-routed via Ollama), with automatic fallback to qwen3.5:9b (fully local).

6.1 The system prompt

The LLM is given a strict role: "Chief Economic Advisor at the International Labour Organization, specializing in AI's impact on labor markets." Rules include: write only in Arabic, stick to the provided numbers (no hallucination), use a critical and objective tone, and follow a fixed 6-section structure (Executive Summary → Labor Market Analysis → Macroeconomic Analysis → Future Scenarios & Risks → Policy Recommendations → Conclusion).

def _format_simulation_summary(results: Dict[str, Any]) -> str:
    summary, report = results.get("summary", {}), results.get("report", {})
    unem, gdp     = results.get("unemployment", []), results.get("gdp", [])
    wages, sectors = results.get("wages", {}), results.get("sectors", {})

    unem_start, unem_end = unem[0], unem[-1]
    unem_peak = max(unem)
    gdp_growth = (gdp[-1] - gdp[0]) / gdp[0] * 100

    wl_start, wl_end = wages["L1_basic"][0],  wages["L1_basic"][-1]
    wh_start, wh_end = wages["L5_expert"][0], wages["L5_expert"][-1]

    lines = [
        f"SCENARIO: {results['scenario'].upper()}",
        f"HORIZON: {report['start_year']} – {report['end_year']}",
        f"AUTOMATION RATE: {report['automation_rate']*100:.1f}% per year",
        "",
        "── LABOR MARKET ──",
        f"Unemployment: {unem_start:.2f}% → {unem_end:.2f}%",
        f"Peak unemployment: {unem_peak:.2f}%",
        "── MACROECONOMY ──",
        f"GDP: ${gdp[0]:.2f}T → ${gdp[-1]:.2f}T  ({gdp_growth:+.1f}%)",
        f"Gini Index: {report['final_gini']:.4f}",
        "── WAGE GAP ──",
        f"Lowest-skill:  ${wl_start:,.0f} → ${wl_end:,.0f}",
        f"Highest-skill: ${wh_start:,.0f} → ${wh_end:,.0f}",
        ...
    ]
    return "\n".join(lines)

§7 Backend: app.py — REST API Surface

FastAPI application exposing 11 HTTP endpoints. All endpoints return JSON. CORS is enabled for all origins so the frontend can run from a separate static file server.

7.1 Endpoint reference

GET/
Health check. Returns API title and version.
GET/api/simulate/{scenario}
Run a full N-year simulation. Path param scenario ∈ {slow, moderate, rapid}. Query params: horizon (default 20), adoption_speed (default 1.0), country (default WLD), export (CSV flag). Also attaches the RandomForest ml_forecast for comparison.
GET/api/compare
Run all 3 scenarios at once. Used by the side-by-side comparison page.
GET/api/baseline
Show the live baseline (employment, workforce, GDP, sector splits) for a country.
GET/api/countries
Returns the 30 supported countries for the frontend dropdown.
GET/api/monte-carlo
Run N Monte Carlo simulations (clamped 100–5000, default 1000) with randomized parameters. Returns mean and 95% confidence band for unemployment and GDP.
GET/api/sensitivity
±20% perturbation analysis on automation_rate and adoption_speed. Reports impact on final-year unemployment.
GET/api/validate
Backtest model against 2000–2020 World Bank unemployment data. Returns MAE and accuracy score.
POST/api/analyze
Body: {scenario, horizon, adoption_speed, country, model}. Runs a simulation, then generates an Arabic economic advisory report via Ollama. Returns the LLM-generated text, token counts, and generation time.
GET/api/models
List Ollama models installed locally. Used by the validation page's model picker.
GET/api/oxford-risk
10-category automation-risk reference (Frey & Osborne 2013, condensed).
GET/api/oxford-full
Curated subset of Frey & Osborne 2013 Appendix A — ~50 occupations across the full probability spectrum, with SOC codes and ranks.

async function apiFetch(path, options = {}) {
  const res = await fetch(API_BASE + path, options);
  if (!res.ok) {
    const err = await res.json().catch(() => ({ detail: res.statusText }));
    throw new Error(err.detail || `HTTP ${res.status}`);
  }
  return res.json();
}

§11 Validation & Accuracy

The project validates the model in three independent ways:

11.1 Historical backtest (/api/validate)

Runs a damped version of the simulation from 2000 to 2020 and compares predicted unemployment to actual World Bank data. The S-curve is multiplied by 0.5 and uses speed=0.8 to reflect that AI adoption was minimal during this period. Reports Mean Absolute Error (MAE) and an accuracy score (max(0, 100 − MAE × 10)).

11.2 Monte Carlo confidence bands (/api/monte-carlo)

Runs 1,000 simulations with randomized parameters:

  • auto_rate ~ Uniform(0.02, 0.10)
  • speed ~ Uniform(0.60, 1.50)
  • Initial total_jobs and workforce jittered by ±5%

Output includes the mean trajectory and the 2.5th / 97.5th percentile bounds — i.e., the 95% confidence interval. Reproducible via np.random.default_rng(42).

11.3 Sensitivity analysis (/api/sensitivity)

Perturbs each parameter by ±20% and measures the change in final-year unemployment. Outputs the impact magnitude for both directions so the user can identify which knobs matter most.

11.4 Cross-validation: macro model vs. ML forecast

Every /api/simulate response includes the ml_forecast field — an independent RandomForest projection. The user can visually compare both lines on the same axis. Divergence is informative: when the macro model and ML model agree, confidence is higher; when they disagree, the macro model's structural assumptions are doing the work.

§13 Model Limitations & Honest Caveats

This is a teaching/exploratory model, not a forecasting tool. The numbers it produces should be understood as illustrating mechanisms, not predicting outcomes.

13.1 Structural assumptions

  • Fixed skill distribution. The 20/30/25/15/10 split across L1–L5 is held constant for 20 years. In reality, education and reskilling shift this distribution.
  • Hardcoded sector sub-ratios. The Services bucket is split Tech/Healthcare/Services-other using fixed 18%/24%/58% ratios, because the World Bank doesn't break Services down further.
  • One job-creation ratio. The assumption that AI creates 0.6 new jobs per 1 destroyed is a single global constant. Different scenarios in literature range from 0.3 to 1.2.
  • No migration, no policy intervention. The model has no government, no UBI, no retraining programs, no minimum wage changes.
  • Linear wage pressure. Wages move proportionally to adoption with a single coefficient per tier — no kinks, no thresholds.

13.2 Forecast caveats

  • RandomForest is trained in-sample. Reported "accuracy" is training accuracy, not held-out. For a serious forecast, time-series cross-validation would be required.
  • The S-curve midpoint of year 10 is an assumption, not derived from data. Real AI adoption could be faster (a few years) or slower (decades).
  • Gini is computed from synthetic incomes, scaled down by a factor of 10⁷ for performance. The ratio is preserved but absolute counts are not real.

13.3 Data caveats

  • World Bank data has 1–2 year publication lag — the "current" year is rarely available.
  • For most countries, SL.EMP.TOTL is missing and employment is derived as labor_force × (1 − unemployment/100), which slightly undercounts subsistence/informal labor.
  • Frey & Osborne 2013 is now 13 years old — automation probabilities for many roles look different today (especially for "creative" work that LLMs have made more automatable than predicted).
