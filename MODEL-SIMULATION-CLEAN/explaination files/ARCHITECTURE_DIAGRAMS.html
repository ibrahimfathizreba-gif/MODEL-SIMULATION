5. run_scenario — What Happens Each Year

The yearly loop. run_scenario calls 6 functions in sequence for each of 20 years (or however long horizon is set to). This sequence diagram shows the order of calls and what data each one produces.

sequenceDiagram autonumber participant U as User / API participant RS as run_scenario participant IS as initial_state participant Loop as Year Loop (i=0..19) participant SY as step_year participant SKB as skill_breakdown participant SCB as sector_breakdown participant WS as wages_by_skill participant GI as gini_index participant SUM as summarize + report U->>RS: run_scenario("moderate", 20, country="USA") RS->>IS: build initial state IS-->>RS: {total_jobs, workforce, ai_adoption, _gdp_base} loop For each year i in 0..19 RS->>SY: step_year(state, i, auto_rate, speed) Note over SY: Updates state in place:
state["ai_adoption"] = logistic(i)
state["total_jobs"] -= lost + created
state["workforce"] *= 1.008 SY-->>RS: {unem, productivity, gdp, spending} RS->>SKB: skill_breakdown(state) SKB-->>RS: {L1: 720M, L2: 1080M, ..., L5: 360M} RS->>SCB: sector_breakdown(state, auto_rate) SCB-->>RS: {Tech: 340M, Mfg: 750M, Health: 440M, Svc: 1.8B} RS->>WS: wages_by_skill(state["ai_adoption"]) WS-->>RS: {L1: $28K, L2: $45K, ..., L5: $244K} RS->>GI: gini_index(wages, skills) GI-->>RS: 0.42 Note over RS: append all values
to results dict end RS->>SUM: summarize(results), report(results) SUM-->>RS: stats + summary RS-->>U: JSON with 20 years of data

6. World Bank Data Fetching Flow

When you ask for a country's baseline, the model fetches up to 7 World Bank indicators (4 for the baseline, 3 for sectors). The first call triggers HTTPS requests; subsequent calls return from the in-memory cache.

Cache-->>GCB: not found GCB->>FWB: fetch_world_bank_baseline("USA") FWB->>WBL: _wb_latest("SL.EMP.TOTL", "USA") WBL->>WB: GET /v2/country/USA/indicator/SL.EMP.TOTL WB-->>WBL: JSON (returns null — only WLD has this) WBL-->>FWB: None FWB->>WBL: _wb_latest("SL.TLF.TOTL.IN", "USA") WBL->>WB: GET .../SL.TLF.TOTL.IN WB-->>WBL: 167_000_000 (labor force) WBL-->>FWB: 167_000_000 FWB->>WBL: _wb_latest("NY.GDP.MKTP.CD", "USA") WBL->>WB: GET .../NY.GDP.MKTP.CD WB-->>WBL: 27_360_000_000_000 ($27.36T) WBL-->>FWB: 27_360_000_000_000 FWB->>WBL: _wb_latest("SL.UEM.TOTL.ZS", "USA") WBL->>WB: GET .../SL.UEM.TOTL.ZS WB-->>WBL: 3.6 (unemployment %) WBL-->>FWB: 3.6 Note over FWB: employment = 167M × (1 − 3.6/100)

8. Scenario Variants — How compare, monte_carlo, sensitivity Differ

Three different ways to use the same engine. compare_scenarios runs 3 fixed scenarios. monte_carlo runs N random variations. sensitivity perturbs each parameter by ±20%.
